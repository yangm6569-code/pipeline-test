# 高校科研管理系统本科答辩 PPT 叙事规划

## 受众与目标
- 受众：本科论文答辩评委、指导教师
- 目标：用 16 页内容清晰说明选题背景、需求分析、总体设计、关键实现、测试效果与后续展望

## 视觉方向
- 风格：深蓝绿色主色的学术型管理系统答辩风格
- 版式：封面与结尾页深色，主体内容页浅色卡片化布局
- 素材：项目内现有流程图、用例图，以及基于真实模块绘制的界面示意

## 16 页结构
1. 封面
2. 目录
3. 研究背景与选题意义
4. 需求分析与建设目标
5. 系统总体架构与技术路线
6. 功能模块设计
7. 角色与权限控制设计
8. 数据库设计与核心实体关系
9. 项目生命周期与流程控制
10. 关键实现一：项目管理与流程详情
11. 关键实现二：审批、报销与成果管理
12. 系统特色与创新点
13. 系统界面展示：登录与工作台
14. 系统界面展示：业务与管理页面
15. 系统测试与实现效果
16. 总结与展望

## 内容依据
- 前端：Vue 3 + Vite + Element Plus + Pinia + ECharts
- 后端：Spring Boot + Spring Security + JWT + MyBatis + MySQL
- 业务模块：项目管理、流程审批、申请审批、成果管理、报销管理、用户角色管理、AI 助手
- 代码证据：`router/index.js`、`permission.js`、`SecurityConfig.java`、`ProjectProcessServiceImpl.java`、各 Controller 与 Entity

## 讲解重点
- 以“项目全生命周期管理”为主线
- 强调“前端路由权限 + 后端 JWT 鉴权”的双层控制
- 重点说明流程推进、待办聚合、经费协同与 AI 扩展能力
