高校科研管理系统 - 全模块操作原型图

这份原型图按你项目代码逐模块绘制，包含“页面线框 + 关键操作流”。每个模块都画出实际可执行操作：查询、重置、导出、新增、编辑、删除、审批、上传、提交、分页、详情查看等。

样式目标：墨刀低保真线框 布局：后台管理系统（顶部 + 右侧导航栏 + 内容区） 模块数：11

0. 全局导航结构

入口路由结构图

本系统管理端界面采用经典的顶栏、主内容区与右侧导航栏组合布局方式，用户登录后可在右侧导航栏进入首页、项目管理、审批管理、成果管理、报销管理、用户管理和角色管理等模块，整体交互路径清晰，符合后台管理系统的操作习惯。

顶栏用于展示系统名称和用户信息，右侧导航栏承担模块导航功能，主内容区用于承载各业务页面与数据表格。用户在任意业务页面均可通过右侧导航快速切换模块，如图4-20所示为系统全局导航原型结构。

权限说明：系统采用基于角色的模块级访问控制，不同角色登录后仅显示其有权限访问的导航菜单。

角色范围：系统管理员（admin）、教师用户（teach）、财务管理员（fAdmin）、一级审批人（oneAuditor）、二级审批人（twoAuditor）、三级审批人（treeAuditor/threeAuditor）。

1. 登录模块

/login

本系统登录页面采用左侧视觉区与右侧登录表单区的双栏布局，整体风格简洁直观。登录表单依次提供用户名输入框、密码输入框和登录按钮，用户完成信息填写后点击登录即可进行身份认证。

系统在登录成功后会根据用户角色自动跳转至默认业务页面，登录失败时给出明确错误提示，确保用户快速定位问题并完成重试，如图4-21所示。

可见权限：未登录用户均可访问本页面。

可操作权限：所有系统用户均可在此输入账号密码完成登录；已登录用户再次访问登录页时将按角色重定向至默认页面。

2. 首页工作台模块

/dashboard

管理端首页采用“统计卡片 + 双栏信息面板”的组织方式。页面上部展示项目总数、进行中项目、待审批和已完成等关键指标，帮助管理人员在进入系统后快速掌握业务总体状态。

页面下部设置最近通知与待办事项板块，用于承接日常管理信息和任务提醒，使用户能够在同一页面完成信息查看与后续操作入口定位，如图4-22所示。

可见权限：系统管理员（admin）、教师用户（teach）。

可操作权限：系统管理员（admin）、教师用户（teach）可查看工作台统计数据与首页信息面板，本页面以查看型操作为主。

3. 项目管理模块

/project/list

项目管理界面采用“查询条件区、数据列表区、分页区”三段式布局。查询区支持按项目名称、项目类型和状态进行组合检索，并提供查询、重置和导出按钮以满足日常数据筛选与留档需求。

列表区以表格形式展示项目核心信息，单行记录支持编辑、删除、报销和详情等操作。页面右上角提供新增项目入口，新增与编辑均通过弹窗完成，并支持预算明细维护，如图4-23所示。

可见权限：系统管理员（admin）、教师用户（teach）。

可操作权限：系统管理员（admin）、教师用户（teach）均可执行查询、重置、导出、新增、编辑、删除、查看详情及进入报销申请等操作。

4. 项目详情 + 报销申请模块

/project/detail/:id

项目详情页面由项目基础信息、预算明细和报销记录三部分构成，用户可在同一页面查看项目全貌及资金执行情况。预算区展示预算总额、已使用金额和剩余额度，便于进行费用控制。

报销记录区提供“申请报销”操作入口，用户可填写报销金额、报销类型、报销描述并上传凭证图片。提交后，报销单进入审批流程，支持在详情弹窗中查看完整审批与附件信息，如图4-24所示。

可见权限：系统管理员（admin）、教师用户（teach）。

可操作权限：系统管理员（admin）、教师用户（teach）可查看项目详情、预算明细、报销记录，并可发起报销申请、上传报销凭证、查看报销详情。

5. 项目流程详情模块

/project/process/:id

项目流程详情页面以“流程阶段标签 + 阶段详情”方式组织信息。用户点击任一流程阶段后，可查看该阶段的状态、审批意见、审批人、提交时间以及已上传附件，便于跟踪流程进度。

对于可提交阶段，页面提供文件上传与阶段提交能力，支持附件查看和下载，形成“上传材料—提交阶段—审批反馈”的完整闭环操作路径，如图4-25所示。

6. 项目流程审批模块

/project/process-approval

项目流程审批界面面向审批角色，页面顶部提供流程阶段、审批状态、审批人及时间范围等检索条件，审批人员可按条件快速定位待处理任务，提高审批效率。

审批列表展示项目阶段、审批意见和提交时间等关键信息。对于“审批中”数据，审批人员可直接执行通过或拒绝操作，并在弹窗中填写审批意见，系统随后刷新审批状态，如图4-26所示。

可见权限：系统管理员（admin）、一级审批人（oneAuditor）、二级审批人（twoAuditor）、三级审批人（treeAuditor/threeAuditor）。

可操作权限：上述角色均可执行查询、分页、审批通过、审批拒绝及填写审批意见等操作。

7. 申请审批模块

/application/approval

申请审批页面用于处理项目相关业务申请，采用检索栏与表格列表组合布局。管理员可按项目名称、申请人、审批状态和当前节点进行筛选，并支持导出当前筛选结果。

在列表中，管理员可查看流程详情与历史审批记录，对待审批申请执行通过或拒绝，并录入审批意见，从而保障业务流程处理的规范性与可追溯性，如图4-27所示。

可见权限：系统管理员（admin）、一级审批人（oneAuditor）、二级审批人（twoAuditor）、三级审批人（treeAuditor/threeAuditor）。

可操作权限：上述角色可执行查询、重置、导出、查看流程详情、查看审批记录、审批通过和审批拒绝等操作。

8. 成果管理模块

/achievement

成果管理界面采用“条件筛选 + 成果列表”的管理模式，用户可按成果名称、成果类型和级别进行快速检索，并可对查询结果进行导出，满足成果数据统计与归档需求。

列表操作支持成果详情查看和成果删除，详情弹窗展示成果所属项目、完成时间及附件信息等字段，方便用户进行成果核对与后续管理，如图4-28所示。

可操作权限：系统管理员（admin）、教师用户（teach）可执行查询、重置、导出、查看成果详情和删除成果等操作。

9. 报销管理模块

/reimbursement

报销管理页面用于集中处理项目报销单据，支持按项目名称、申请人、报销类型和审批状态进行筛选，并提供导出功能以便财务留档与台账管理。

10. 用户管理模块

/user

用户管理界面采用左侧部门树与右侧用户列表的双栏布局。管理员可在部门树中执行新增、编辑、删除操作，并通过点击部门节点对用户列表进行快速过滤。

用户列表支持新增、编辑、删除和导出操作。新增或编辑用户时可维护账号、角色、部门、联系方式和状态等基础信息，实现人员与权限数据的一体化维护，如图4-30所示。

可见权限：系统管理员（admin）。

11. 角色管理模块
